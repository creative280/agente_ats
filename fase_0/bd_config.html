Conexion de base de datos

Cambia solo el nombre de la base de datos, manteniendo el mismo servidor y usuario de DATABASE_URL.
Ejemplo: cambiar de cifras_channel a channel.

Descripcion general de la BD

Describe en una frase de que trata tu base de datos. Esto le da contexto general al agente.
Ejemplo: "Base de datos de gestion comercial con informacion de clientes, contratos de venta a credito y cartera de cobros."

Tablas con descripcion 0

Selecciona las tablas mas importantes y agrega una descripcion en lenguaje natural. No necesitas describir todas, solo las que mas uses.
El agente usa estas descripciones para elegir las tablas correctas cuando le haces una pregunta.

Ejemplo: Para la tabla cartera, escribe: "Saldos pendientes por cobrar de cada contrato. Incluye cuotas, mora, fecha de pago."
Haz clic en una tabla para agregarla:
Haz clic en las tablas de arriba para empezar a describirlas.

Ejemplos de consultas SQL 0

Agrega ejemplos de preguntas y las consultas SQL que las responden correctamente. Esto le ensena al agente como se relacionan tus tablas y que convenciones usa tu BD.

Tip: Incluye consultas con JOINs entre tablas, funciones de agregacion, y filtros comunes. Mientras mas ejemplos, mejor sera el agente generando SQL.

Ejemplo:
Pregunta: "Top 10 clientes con mas deuda"
SQL: SELECT c.nombre, SUM(ca.saldo) as deuda FROM clientes c JOIN cartera ca ON c.id = ca.id_cliente GROUP BY c.nombre ORDER BY deuda DESC LIMIT 10
Aun no hay ejemplos. Agrega tu primera consulta de ejemplo.

Reglas de negocio 0

Anota convenciones o reglas especiales de tu BD que el agente necesita saber.

Ejemplos utiles:
• estado = 'A' significa activo, 'I' inactivo
• Las fechas en la tabla contrato estan en formato YYYY-MM-DD como VARCHAR
• tipo_cliente = 1 es persona natural, 2 es empresa
• La tabla cartera tiene una fila por cuota, no por contrato
• Montos en valor_cuota incluyen IVA
